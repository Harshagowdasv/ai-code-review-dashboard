2) High-Level Architecture
Three-Tier Architecture
Frontend (React)

Single Page Application with client-side routing
Manages UI state with React hooks and Context API
Makes async API calls to backend
Displays code review results with syntax highlighting

Backend (Express)

RESTful API with clear resource endpoints
Handles authentication/authorization (JWT)
Processes code submissions
Integrates with OpenAI API for code analysis
Implements rate limiting and caching

Database (MongoDB)

Stores user accounts and authentication data
Persists code reviews and AI feedback
Tracks analytics (review counts, languages, issue types)
Flexible schema for different code review formats

Data Flow

User submits code → React form validation
Frontend sends POST request with code snippet
Express middleware validates & authenticates
Controller calls AI service for analysis
AI service makes OpenAI API call
Response parsed and stored in MongoDB
Formatted data returned to frontend
React updates UI with review results

Key Design Decisions

Separation of Concerns: Controllers handle HTTP, services handle business logic
Service Layer: Isolates AI integration for easier testing/mocking
Error Boundaries: Graceful error handling on both client and server
Modular Routes: Each resource has its own route file
Environment Config: All secrets in .env files (never committed)

Interview Talking Points
Scalability:

Service layer makes it easy to swap AI providers
MongoDB's flexible schema handles different code languages
Middleware pattern allows easy addition of features (rate limiting, logging)

Security:

JWT authentication with httpOnly cookies
Input validation on both client and server
Rate limiting to prevent API abuse
Environment variables for secrets

Performance:

Code splitting in React for faster initial load
Database indexing on frequently queried fields
Optional caching layer (Redis) for repeated reviews
Debounced search inputs

Best Practices:

RESTful naming conventions (GET /api/reviews, POST /api/reviews)
Consistent error response format
Centralized error handling
Modular, testable code structure

This architecture demonstrates production-ready patterns that scale beyond a simple CRUD app. Perfect for portfolio discussions!
